## 0.9.26

镜像 tag 跟随主项目升级 0.9.25 → 0.9.26。加载项选项、端口、卷挂载**均无变化**。

**紧急修复版：115 云盘上传全面失败。使用 115 备份的用户请尽快升级。**

- **修复 115 云盘上传全面失败**：受网盘服务端策略调整影响，此前版本已无法把录像上传到 115 云盘，而且**没有任何提示**——录像照录、回放照看，只是默默不再上云。本版恢复上传，并让云瞰能自动适应网盘后续的同类调整。
- **云备份中断时会主动通知你**：上传持续失败超过一段时间会推送提醒。以前上传停了没有任何症状，往往要等磁盘写满、旧录像被自动清理，才发现云端早就缺了好几天。
- **新增「云备份状态」面板**：设置 › 115 云盘直接显示最近一次成功上传的时间、还有多少段录像在排队等待上传；停摆时整张面板转为醒目的警告。
- **推送语言跟着手机走**：手机 App 设成哪种语言，推送通知就用哪种语言，家里几台手机各说各的语言互不影响。以前所有设备统一使用服务器上设置的那一种语言。服务器上的「推送语言」设置现在只作为老版本 App 的兜底。
- **安卓 App 修复**：点过推送通知之后再切换应用语言，可能闪退或反复弹出旧事件详情（需同步升级到 0.9.26 版 App）。

## 0.9.23

镜像 tag 跟随主项目升级 0.9.22 → 0.9.23。加载项选项、端口、卷挂载**均无变化**。

本次核心是厂商相机深度接入 + 磁盘清理重做 + 检测可靠性提升：

- **深度接入海康 / 大华 / Reolink 厂商相机**：相机自带的智能事件（有人 / 车辆 / 越界 / 镜头被遮挡破坏）可直接接入云瞰，和本地 AI 一样能推送、能触发自动化；还能用上相机自带的白光补光灯、声光报警、云台连续转动与预置位、对讲喊话。添加时新增「测试连接 / 探测能力」按钮，提交前即可看到能识别出哪些功能。
- **白光补光灯一键开关 + 声光 / 灯光自动化**：支持的厂商相机在实时画面 / 常用面板（手机与网页）一键开关白光补光灯；自动化可把「开灯」「声光报警」设为动作并指定自动关闭时长，夜间有人自动亮灯、鸣笛威慑。
- **相机事件更聪明（按需识别人脸 / 车牌）**：厂商相机报「有人 / 有车」时云瞰顺手做人脸 / 车牌识别——认出熟人单独记一条人脸事件（可单独推送 / 自动化），识别到的车牌补充到对应车辆事件。只在事件瞬间识别，不额外占资源。
- **厂商相机画面更快更清**：缩略图与事件抓拍直接向相机取图，出图更快、全分辨率；部分高清（H.265）相机不再「半天刷不出 / 抓到空白」，在线状态判断更准。
- **磁盘清理更省心、更安全**：清理设置简化为「录像总容量上限 + 整盘占用保护」两个主开关，写满自动覆盖最旧的（像监控硬盘），彻底解决本地录像无限堆积撑满盘的问题；盘被云瞰以外的数据占满时绝不误删你的录像、只提醒。
- **暗房进人 / 开灯瞬间不再漏检**：修复画面剧烈变化那两三秒 AI 短暂「失明」漏掉关键一刻的问题。
- **本地空间更省**：仅在有事件时才录像的相机，其「无事件缓冲录像」保留从 1 小时收紧到 5 分钟。
- **对讲 / 喊话更易接通**：修复部分海康相机喊话 / 对讲偶发「第一次失败、第二次才成功」。
- **推送语言修复**：修复部分中文用户登录后误收英文推送；推送语言以「设置 → 通知」里手动选择为准。
- **其它**：长时间运行更稳定（后台识别内存占用优化）；手机端云台面板小屏布局更合理；新增「镜头遮挡破坏」事件在手机端的展示。

- 通过 `devices: /dev/dri:/dev/dri:rwm` + `udev: true` 透传 Intel iGPU render
  node，让 OpenVINOExecutionProvider 能跑 GPU 后端。
- runtime 选择走 backend 自动降级链（tensorrt → cuda → openvino → cpu），
  openvino 镜像里 `onnxruntime-openvino` 已装、cuda / trt wheel 不在，自然
  落到 OpenVINO，无需注入 env。`detection.device_type` 沿用镜像默认 `AUTO`。
- 端口段与 CPU 加载项一致（23406 / 23880 / 24214 / 24215 / 23515），不能与
  CPU 加载项**同时**启用。
