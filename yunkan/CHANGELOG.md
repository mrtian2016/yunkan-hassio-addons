# Changelog

## 0.9.20

镜像 tag 跟随主项目升级 0.9.19 → 0.9.20。加载项选项、端口、卷挂载**均无变化**。

本次核心是实时检测框重做 + 回放与录像升级：

- **实时画面检测框更流畅（三端）**：实时监控的检测框改由 App / 网页实时绘制，根治部分摄像头上的卡顿 / 画面出不来；框跟手更稳、不再忽大忽小，跌倒等告警用红框标出。
- **回放升级为「NVR 多路同步」（网页）**：多路摄像头按同一时间轴一起播放 / 拖动 / 倍速，单路卡顿不拖累全局；时间轴标出有录像 / 空档。
- **新增「文件管理」（后台）**：统一管理录像与事件快照——按相机 × 日期浏览、看大图 / 播分段、查各相机占用，并支持物理删除（同时清本地 + 115 网盘）。
- **录像更可靠**：加固部分摄像头（音频特殊）录像可能悄悄中断 / 变 0 时长——异常时自动降级保画面；新增「录像健康监测」，该录没录会在后台首页标红。
- **移动端也能看「鸟瞰」总览**：Android / iOS 加入鸟瞰墙，横屏沉浸全屏 + 捏合缩放 / 拖动 / 双击，自动选最顺畅播放方式。
- **事件快照可一键隐藏检测框（三端）**；**实时监控音量滑块 + 侧栏可折叠（网页）**；**设置项标出出厂推荐值（网页）**；**可自动清理零散录像（网页，默认关）**。
- **智能识别总开关支持待机省资源**：关闭后真正释放显卡 / 模型不再空跑（此前开关未真正生效），重新打开约数秒恢复。
- **一批修复**：「区域无人 X 时长通知」连发多条；个别（未开检测 / 录像）摄像头误判离线；升级时偶发数据库繁忙报错；鸟瞰播几秒卡住 / 横屏被裁；iOS 实时检测框不显示。

## 0.9.18

镜像 tag 跟随主项目升级 0.9.17 → 0.9.18。加载项选项、端口、卷挂载**均无变化**（加载项本就以 host 网络运行，HomeKit 配对所需的局域网发现开箱即用）。

本次核心是接入 Apple「家庭」+ 检测交互升级：

- **接入 Apple「家庭」（HomeKit，Pro）**：摄像头加进 iPhone / iPad「家庭」App——实时查看、Siri 唤出、运动录像存进 iCloud（HomeKit 安全录像），并支持收听摄像头声音 + 手机麦克风对讲；后台扫码配对，每台单独开关。
- **进出方向识别升级为「画一条线」**：门口 / 过道画一条线，目标越过即判定进 / 出，比画两个区域更准更省事；自动化可指定「越过哪条线」触发。已有配置自动迁移。
- **实时监控自定义视频墙版面**：在内置宫格之外自由拖出任意版面（拖放分配、缩放占大格、多套保存、随账号跨设备同步）。
- **自动化更聪明**：选好摄像头后区域 / 绊线自动收窄到这些相机；已停用的摄像头不再参与自动化。
- **可自选要检测的目标类别**：可指定只检测你关心的类别（人 / 自行车 / 汽车 / 摩托 / 公交 / 卡车 / 鸟 / 猫 / 狗共 9 类），全局设一套并可给每台摄像头单独收窄；只想要「人 + 车」就关掉鸟，不再被飞鸟刷屏。（关「人」会连带停用人脸 / 姿态识别，关「车辆」连带停用车牌识别。）
- **云台光学变焦（手机端）**：实时预览云台盘加放大 / 缩小，点按走一档、按住连续，真·光学变焦。
- **更多体验**：回放日历标出有录像的日子；手机端记住上次静音选择；可手动指定 AI 加速硬件（CPU / Intel 核显 / NVIDIA 显卡 / 极速）。
- **NVIDIA 显卡极速加速（TensorRT）修复可用性**：此前加速页可能误报「未检测到显卡」让有显卡的用户也开不起来极速加速，现已修复，且加速过程中模型一完成即时显示「已加速」。
- **性能监控支持 Intel 核显**：性能页新增 Intel 核显的型号 / 占用率 / 共享显存 / 频率，与 NVIDIA 显卡并列，Intel 平台（N100 / J4125 等）用户也能看 AI 占用；未触发的检测器行也显示相机名。
- **新增「升级残余清理」**：一键回收旧版本镜像、历史数据库备份等，安全释放磁盘（自动保留当前版本 + 回滚版本）。
- **隐私与安全增强**：非管理员不再能看到摄像头账号密码；日志 / 告警里的密钥打码。
- **多语言修复**：非中文系统默认英文；离线推送通知按 App 内所选语言显示。
- **一批修复**：提升小目标（小猫、小狗等）检测；修复个别摄像头录像可能异常占满磁盘（并自动清理历史残留）；满屏忽略区贴边漏报等。

## 0.9.17

镜像 tag 跟随主项目升级 0.9.16 → 0.9.17。加载项选项、端口、卷挂载**均无变化**。

本次核心是 AI 误报治理 + Home Assistant 联动升级：

- **AI 检测误报系统性治理，误报大幅减少**：① 重做「画面变化」检测——光线渐变、开灯、红外切换、闪电等场景自动识别并跳过，飞鸟掠过等一闪而过的干扰不再生成事件；② 停在画面里的车辆 / 物体不再被反复识别报警，全程至多提醒一次；③ 同类事件「连珠炮」收敛——同一摄像头同类事件默认 60 秒内只报一条（可调）。
- **Home Assistant 联动升级**：接入 MQTT 后，每台摄像头新增「实时直播」与「事件快照」——在 HA 面板里直接看云瞰直播画面，检测事件的快照（带检测框）也能作为图片实体展示；MQTT / Webhook 推送支持插入快照链接。
- **新增「鸟瞰」多画面总览**：所有摄像头自动拼成一幅画面，打开一路流纵览全家，省流量省性能。
- **新增「进出方向识别」**：门口画两个检测区域，按先后顺序判断「进门 / 出门」，搭配「有人回家时通知」模板一步配好回家提醒。
- **摄像头支持一键停用 / 启用**：停用即停画面、录像、检测，历史事件收起；重新启用原样恢复。
- **跳转回放正好落在事件发生前**：修复系统繁忙时事件时间偏晚，且跳转回放提前 5 秒起播。
- **系统更新更小更快**：AI 语义模型改为按需下载（不再内置镜像），镜像瘦身数 GB，加载项日常更新下载量显著减少。
- **其它修复**：摄像头快照偶发花屏；没开检测 / 录像的摄像头被误判「离线」；国标（GB28181）摄像头直播 / 录像无声与周期性卡顿；只开「哭声检测」时状态不显示运行中；存储占用统计改为按录像目录真实占用；手机端实时 / 快照画面比例修复（非 16:9 摄像头不再变形）；长期运行后直播临时文件可能残留、缓慢占用磁盘的问题已根治（异常断电 / 重启后也会自动清理）。

## 0.9.16

镜像 tag 跟随主项目升级 0.9.15 → 0.9.16。加载项选项、端口、卷挂载**均无变化**。

本次核心是 AI 检测实时性根治：

- **修复多台摄像头同时 AI 检测时检测明显「慢半拍」**：摄像头较多时，AI 检测可能落后实际画面几十秒 —— 事件中心里事件要等约 30 秒才冒出来，从事件点「跳转回放」也会停在事件之后、错过关键画面。现已根治，检测恢复实时：事件几乎立即出现，跳转回放也能精准停在事件发生那一刻。摄像头越多，改善越明显。
- **进一步修复部分 H.265（HEVC）相机的黑屏 / 无声**：少数 H.265 相机在特定情况下直播黑屏或听不到声音，现已修复。
- **对讲更稳**：修复对讲偶发的连接失败。
- **网页端云台操作升级**：网页直播支持群晖 NVR 风格的云台操作（滚轮变焦、按住方向键转动云台），设备管理直播弹窗支持主 / 子码流切换。
- **手机端回放支持横屏全屏 + 双指缩放 / 拖动；登录支持只填公网地址即可**（安卓 / iOS）。
- **系统更新显示下载进度**：在线更新时新增实时下载进度条，下载到哪一步、下了多少一目了然，不用再盯着空白干等。
- **其它优化**：移除手机端一个无效的运动检测开关；缓解偶发的数据库占用导致的卡顿。

## 0.9.0

跟随主项目跨 minor 版本一次升级(0.7.8 → 0.9.0)。**建议升级前先备份 HA 配置**
(Settings → Add-ons → 备份)。

  (build.yaml + README 同步)。
- 主项目 0.7.8 → 0.9.0 用户可见变化(详见主项目 CHANGELOG):
  - **磁盘自动清理**:新增 `cleanup` 独立服务,超过保留天数 或 磁盘占用超阈值
    时自动清理已上传/已删除录像,可配 `dry_run` 空跑模式。
  - **License 升级**:支持 Apple IAP 订阅(iOS)与 self-host 卡密双 source 合并。
  - **直播鉴权重构**:mediamtx HLS/WebRTC 端口绑 127.0.0.1,所有外部播放走
    nginx 反代 + JWT 短 token,客户端 80% TTL gapless 续期(看 Live 无感知)。
  - **摄像头管理重设计**:Cameras 页 + 添加流程 + 直播弹窗整套重做;add 后台
    立即抓首帧,免去等 snap 进程 30s 才出缩略图。
  - **Web Admin 服务管理**:跨进程重启 detection / automation / uploader / snap
    / postprocess / cleanup,无需重启整个加载项。
  - **VAAPI 解码容错**:对 H.265 摄像头注入 `+discardcorrupt` 丢损坏帧,
    Intel iGPU 解码 HEVC 不再因 B/P 帧引用缺失整路掉线。
- 端口、卷挂载、加载项选项**均无变化**(仍只有 `timezone` 一项)。

## 0.7.3-1

- 端口段二次偏移避开 HA Core 内置 go2rtc 占用的 18554/18555。所有端口在原
  `+10000` 偏移基础上再 `+5326`：
  - HTTP `18080 → 23406`
  - RTSP `18554 → 23880`
  - HLS `18888 → 24214`
  - WebRTC WHEP `18889 → 24215`
  - WebRTC ICE UDP `18189 → 23515`
  - 容器内 loopback 端口（api 18000、web 13000、redis 16379、mediamtx api
    19997 等）同步偏移，避免 `host_network` 模式下与 HA 自身共享 loopback 撞车。
- 加载项 Dockerfile 用 sed 在镜像基础上 patch `/etc/skyview/mediamtx.yml` /
  `/app/config/config.toml` / `/etc/nginx/conf.d/skyview.conf` / s6-rc.d 配置，
  并覆盖 `SKYVIEW_PUBLIC_PORT` 与 `HEALTHCHECK`。
